# Discovery: organvm/universal-node-network

**Discovered:** 2026-06-22 | **Verdict:** PROMOTE

## Value Thesis

Universal Node Network contains two distinct working assets that justify ranked promotion. The first is a node/network/discovery layer — `Node`, `Network`, and `DiscoveryService` in Python — providing capability-based routing, topology management, and TTL-aware peer discovery. This is Phase 1-2 infrastructure: the data model and in-memory registry are solid, but the GitHub Actions dispatch transport described in the README has not yet been implemented. The second, and higher-value, asset is the **lens assembly system**: `src/hierarchy.py` encodes 21 philosophical and scientific lenses (mathematics, thermodynamics, ecology, governance, the technium, chaos, the noosphere, and 14 others) organized into 7 Unix-path-named strata, and `src/assembly.py` provides a fully functional `assemble(task_description)` function that selects 2-3 contextually appropriate lenses, generates structured insights (what each lens adds, strips, and critiques), and detects inter-lens conflicts using three canonical resolution patterns. Combined with `src/homeostasis.py` — a `HomeostasisMonitor` pre-wired with 7 ORGANVM vital signs (active repo count, stale repo ratio, governance-to-product ratio, infra WIP count, test coverage, omega velocity, entropy rate) each carrying automatic corrective actions — the repository constitutes a working **autonomic reasoning and health layer** for the entire ORGANVM estate. Any organ's agent can `from universal_node_network.assembly import assemble` to get structured multi-lens analysis; any conductor can instantiate `HomeostasisMonitor` with `create_organvm_vitals()` to get threshold-triggered corrective responses rather than passive monitoring. The single best first task is to expose `assemble()` and `create_organvm_vitals()` as a published CLI (`unn assemble "..."` → JSON output) so any ORGANVM repo can use the lens system without importing the full network layer as a dependency.
